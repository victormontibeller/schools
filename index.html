Arquitetura do Sistema

O sistema deverá utilizar uma arquitetura de Monólito Modular, permitindo evolução contínua sem a complexidade de microserviços. Cada domínio deverá possuir seu próprio módulo independente.

Traefik

HTTPS · Reverse Proxy · Load Balancer

Django (Monólito Modular)

Tenant Middleware → Auth → Service → View

PostgreSQL + django-tenants

Schema por escola · Isolamento total

Celery + RabbitMQ

E-mail · WhatsApp · Relatórios · IA

Grafana · Prometheus · Loki

Logs JSON · Correlation ID · Métricas

Stack Tecnológica

Cada tecnologia deverá ser utilizada dentro de seu papel específico e intransferível.

Backend
Python 3.13 Django Django Templates HTMX Alpine.js Bootstrap

Toda lógica de negócio permanecerá no backend. React, Angular ou Vue não serão utilizados no MVP.

Banco de Dados
PostgreSQL django-tenants

Um Schema PostgreSQL por escola (Tenant). Nenhum compartilhamento de dados entre Tenants.

Processamento Assíncrono
Celery RabbitMQ (Broker)

E-mail · WhatsApp · Relatórios · Processamentos demorados · Tarefas de IA. Redis não será o Broker.

Cache
Redis

Cache de consultas · Dashboards · Sessões · Rate Limiting.

Infraestrutura
Docker Docker Compose Docker Swarm Traefik

Traefik: Reverse Proxy · HTTPS · Load Balancer · Roteamento por domínio.

Observabilidade
Grafana Prometheus Loki OpenTelemetry

Logs JSON · Correlation ID por requisição · Métricas desde a Sprint 0.

Regras de Engenharia

É Obrigatório
Black + Ruff + PEP 8
Type hints em todas as funções públicas
Toda regra de negócio em services/
Logs estruturados em JSON com Correlation ID
Auditoria em toda operação de escrita
Testes para toda regra de negócio crítica
Soft Delete obrigatório em todas as entidades
ADRs para decisões arquiteturais
DRY e SOLID em todo o código
É Proibido
Regras de negócio em Views
print() para logging
SQL direto em Views
Duplicação de código
Dependências circulares entre módulos
Ignorar o contexto do Tenant
Remover registros fisicamente
Secrets em código-fonte
Informações sensíveis em logs
